## 一、大规模 Benchmark 实验

### 实验规模
- **65 个实验**，5 种方法 × 2 数据集 + 逐对话拆分 + 参数搜索 + 学习曲线 + DialSim 复现
- **2446 题**：LoCoMo 1976 题 + LongMemEval 470 题
- **耗时 ~3.5 小时**，~52,000 次 DeepSeek API 调用
- Embedding: qwen3-embedding via LAN (192.168.1.100:11435)
- LLM Judge: DeepSeek-chat API

### 1.1 主实验结果（3 轮）

| 方法 | LoCoMo P@1 (1976题) | LongMemEval P@1 (470题) |
|------|:---:|:---:|
| Cosine | 22.4% | 41.7% |
| BM25 | **30.9%** | 48.3% |
| DGD+GT | 26.6% | **50.0%** |
| DGD+Judge | 24.1% | 48.3% |
| DGD+CMS | 23.6% | 46.2% |

**发现：** DGD 在语义检索任务（LongMemEval）上显著优于 baseline（+8.3pp vs cosine），但在关键词匹配型任务（LoCoMo）上不如 BM25。

## 二、进化搜索实验

### 2.1 Judge Prompt 进化 V1（usage-based）

**设置：** 10 种群，15 代 GA 进化，200 评测样本（100 LoCoMo + 100 LongMemEval）
**评测方式：** 模拟 agent 回答 → Judge 判断"用了哪个记忆" → 和 GT 对比
**LLM：** DeepSeek-chat（变异 + 评测）

| 阶段 | best accuracy | avg accuracy |
|------|:---:|:---:|
| 原始 baseline (seed 0) | 28.9% | - |
| 最优种子 (seed 4) | 43.1% | - |
| Gen 0 (初始化) | 43.1% | 34.8% |
| Gen 5 | **66.7%** | 48.1% |
| Gen 15 (最终) | **66.7%** | 54.0% |

**进化谱系：**
```
seed-004 (43.1%) "specific claim, fact, or detail ONLY found in that fragment"
  → gen004-000 (61.1%) + "rigorous" 判断标准
    → gen005-006 (66.7%) + "genuinely and exclusively referenced"
```

### 2.2 Judge Prompt 进化 V2（relevance-based）

**设置：** 同 V1，但改变评测目标——不模拟 agent，直接问"哪个记忆和答案相关"
**关键改进：** 每个评测样本只需 1 次 API 调用（V1 需要 2 次）

| 阶段 | best accuracy | avg accuracy |
|------|:---:|:---:|
| Gen 0 (初始化) | 78.6% | 56.0% |
| Gen 6 | 88.9% | 84.4% |
| Gen 8 | **90.9%** | 86.5% |
| Gen 15 (最终) | **90.9%** | 90.0% |

**最优 prompt 进化出的 7 条规则：**
1. DIRECT ANSWER REQUIREMENT — 必须直接包含答案
2. SELF-CONTAINED COMPLETENESS — 单个片段就能回答
3. NO COMBINATION ALLOWED — 不允许组合多片段
4. NO TOPICAL FALLACY — 话题相关不算
5. NO PARTIAL CREDIT — 不给部分分
6. VERIFIABLE INFORMATION — 必须可验证
7. STRICT INTERPRETATION — 有疑问就排除

### 2.3 FunSearch DGD Update Rule（代码进化）

**设置：** FunSearch 群岛模型，5 岛，100 迭代，2 样本/迭代，500 题 GT benchmark
**进化目标：** `dgd_update(M, key, target, dim, alpha, eta)` 的 Python 函数体

| 指标 | 值 |
|------|-----|
| 总评测程序 | 43 |
| 有效程序 | ~21% 成功率 |
| Baseline P@1 | 26.0% |
| 最终 best P@1 | **31.2%** |
| 提升 | +5.2pp |

**最优进化规则的两个创新：**
1. `delta = error / key_norm2` — 误差除以 key 范数归一化
2. 非对角线去掉 alpha 衰减 — 只有对角线有 forget 项

**对比：** η=0.05 参数搜索达到 34.4%，比 FunSearch 最优（31.2%）更高。说明搜索空间可能太小。

### 2.4 进化搜索核心发现

1. **评测目标比进化策略更重要**：V2 换问法（usage→relevance），起步 78.6% 就超过 V1 终点 66.7%
2. **Prompt 措辞影响巨大**：不进化，光换种子 prompt 就 +14pp（28.9% → 43.1%）
3. **进化有效但会饱和**：V1 Gen 5 到顶，V2 Gen 8 到顶，之后 avg 涨但 best 不动
4. **LLM 自动发现的规则比人写的好**：进化出 7 条精确规则 vs 人写的 4 条模糊规则
5. **代码搜索空间小**：DGD update rule 太简单（~15 行），FunSearch 能改进但有限

### 2.5 过拟合风险

⚠️ **所有进化实验的评测和选择用了同一组 200 个样本。** 90.9% 的 V2 结果可能虚高。

待做验证：
- Hold-out：在剩余 1776 题（LoCoMo）上测试
- Cross-dataset：在 DialSim（完全不同的数据集）上测试
- 如果 90% → 80%+ 是真泛化，掉到 60% 是过拟合

## 六、FunSearch v4 — 类进化 + 多模型（最终阶段）

### 6.1 架构升级

从进化单个函数扩展到进化整个 `AssociativeMemory` 类：

| 版本 | 进化目标 | 搜索空间 | 最优 P@1 |
|------|---------|---------|---------|
| v1 | `dgd_update()` 函数（15 行） | 小 | 31.2% (3轮) / 38.2% (10轮) |
| **v4** | **整个类（init+query+update+自定义状态）** | **大** | **79.4% (10轮)** |

关键改进：
- 类进化：LLM 可以发明 momentum buffer、history、Adam 状态等
- 多模型混合：9b(60%) + DeepSeek(30%) + GPT-5.4(10%)
- 10 轮评测（vs 之前 3 轮），区分度大幅提高
- numpy 加速评测（cosine 比较提速 3-5x）
- 5 分钟超时（自动淘汰过度复杂的类）
- 部署到 5070 服务器无限运行

### 6.2 进化历程

| 迭代 | P@1 | 模型 | 发明了什么 |
|------|-----|------|-----------|
| seed | 26.8% | baseline | 标准 Hebbian |
| gen001 | 61.2% | GPT-5.4 | 32 槽记忆缓存 + 门控融合 + 统计归一化 |
| gen005 | 72.4% | 9b | 零初始化 M + Hebbian 强化 + 动量 |
| **gen019** | **79.4%** | **DeepSeek** | **Adam 优化器 + 非线性 query + 自适应学习率 + 历史一致性奖励** |

### 6.3 最优算法分析（79.4%, DeepSeek 生成）

核心创新对比：

| 组件 | 原始 DGD | 进化版 79.4% |
|------|---------|-------------|
| M 初始化 | Identity（直通） | **Zero（白纸）** |
| 学习信号 | -η·error·key | **-η·error·key + η·target·key（双倍强化）** |
| 优化器 | 无 | **Adam（beta1=0.9, beta2=0.999）** |
| 动量 | 无 | **velocity matrix, momentum=0.9** |
| 学习率 | 固定 η | **η / (1 + avg_error)（自适应）** |
| query | M @ key | **M @ key + 0.1 × 非线性项（key²）** |
| 历史 | 无 | **100 步 key/target 缓冲，一致性奖励** |
| 防爆 | 无 | **clip(M, -10, 10)** |

### 6.4 全量验证

## 八、LongMemEval 跨数据集泛化验证

### 8.1 实验目的

验证在 LoCoMo 上进化出的算法（gen057, 87.6%/500q）是否能在**从未见过**的 LongMemEval 数据集上泛化。

### 8.2 结果

| 轮数 | P@1 | P@3 | P@5 |
|:----:|:---:|:---:|:---:|
| 1 | 63.6% | 82.6% | 90.0% |
| 3 | 81.5% | 94.9% | 97.7% |
| 5 | 86.2% | 97.7% | 98.7% |
| **10** | **92.3%** | **98.9%** | **99.6%** |

### 8.3 对比

| 方法 | LongMemEval P@1 | 备注 |
|------|:---:|------|
| Cosine baseline | 41.7% | 无学习 |
| BM25 | 48.3% | 关键词匹配 |
| DGD-GT 标准 Hebbian 3轮 | 50.0% | 原始算法 |
| DGD-GT 标准 Hebbian 10轮 | 56.0% | 原始算法 |
| **进化版 10轮** | **92.3%** | **+36.3pp vs 原始，+50.6pp vs cosine** |
| Hindsight+TEMPR (SOTA) | 91.4% | GPT-4o + 多策略（端到端，非纯检索） |
| EverMemOS (SOTA) | 93.0% | 商业系统（端到端） |

### 8.4 意义

1. **真泛化**：算法在 LoCoMo 上进化，从未见过 LongMemEval，直接达到 92.3%
2. **接近 SOTA**：92.3% 接近 Hindsight+TEMPR (91.4%) 和 EverMemOS (93.0%)
3. **成本差异巨大**：SOTA 系统用 GPT-4o + 知识图谱 + 多策略融合，我们只用一个 256×256 矩阵
4. **P@5=99.6%**：前 5 个检索结果几乎必含正确答案

### 8.5 注意

- LongMemEval oracle 版只给相关 session（无干扰），比真实场景容易
- SOTA 系统报的是端到端准确率（检索+生成），我们报的是纯检索 P@1
- 不完全可比，但量级上已经在同一水平

## 九、过拟合检测与自适应评测集（晚间发现）

### 9.1 过拟合事件

进化到 gen151 时，500 题分数从 87.6% 涨到 88.4%，看似进步。但全量验证揭示了真相：

| 算法 | 500 题 | 全量 1976 题 | 膨胀比 |
|------|:---:|:---:|:---:|
| gen057 (gpt54) | 87.6% | **49.5%** | 1.8x |
| gen151 (deepseek) | 88.4% | **30.1%** | **2.9x** |

**500 题上"进步"了 0.8pp，全量上退步了 19.4pp。** 进化选了一个在 500 题上过拟合的算法当冠军。

### 9.2 原因分析

- 500 题评测集太小，无法区分"真正更好"和"记住了这 500 题的特征"
- 之前设的自动扩大阈值是"500 题 >90% 才切"，但 88.4% 就已经严重过拟合了
- 膨胀比从 1.8x 跳到 2.9x 是明确的过拟合信号

### 9.3 修复：基于膨胀比的自适应评测集

将自动扩大逻辑从"固定阈值"改为"过拟合比率"：

```
之前: 500 题分数 >90% → 扩到 1000 题
现在: 膨胀比 >2.0x → 扩到 1000 题
      膨胀比 >1.8x → 扩到 1500 题（在 1000 题上）
      膨胀比 >1.5x → 扩到全量（在 1500 题上）
```

### 9.4 天花板测试（87.6% 算法，全量 1976 题）

| 轮数 | P@1 | P@3 | P@5 | 每轮增量 |
|:----:|:---:|:---:|:---:|:---:|
| 1 | 23.1% | 37.3% | 44.8% | - |
| 3 | 38.7% | 57.8% | 67.3% | +5.2pp/轮 |
| 5 | 44.2% | 63.9% | 71.8% | +2.8pp/轮 |
| 10 | 49.5% | 69.3% | 76.1% | +1.1pp/轮 |
| 15 | 52.2% | 70.7% | 77.4% | +0.5pp/轮 |
| 20 | 53.3% | 71.5% | 78.2% | +0.2pp/轮 |
| 30 | 54.0% | 72.0% | 78.8% | +0.07pp/轮 |

**天花板约 55%。** 20 轮后增速 <0.2pp/轮，基本饱和。

### 9.5 关键教训

1. **500 题分数不可信**——必须有全量验证交叉检查
2. **膨胀比是过拟合的早期预警**——从 1.8x 跳到 2.9x 是红灯
3. **"分数涨了"不等于"算法变好了"**——进化也会过拟合
4. **天花板 ~55%** 是当前搜索空间 + 数据集的真实上限
