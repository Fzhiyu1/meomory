问题：RAG 不是记忆

当前 LLM agent 的"记忆"本质上是 RAG（向量检索）——一个图书馆。它不学习、不遗忘，同样的查询永远返回同样的结果。

真正的记忆应该：被使用的关联变强，不用的自然衰减，无需主动搜索就能浮现。

方法：DGD + LLM Judge 自动反馈

用户消息 → DGD 匹配记忆 → 注入上下文 → LLM 回复 ↓ LLM Judge 判断引用了哪些 ↓ DGD 强化被引用的关联 → 下次排序改变

关键组件

DGD 从 Hope 架构（NeurIPS 2025）拆出的权重更新规则，做独立外挂记忆模块（原创用法）

LLM Judge 观察 agent 回复，判断哪些注入记忆被实际引用，产生自动学习信号

无人工标注 整个闭环全自动，agent 不知道自己在被观察

反馈机制演进

版本方法反馈准确率DGD 学习效果
v0人工标注100%✅ 有效但不可扩展
v1Embedding 相似度~50%❌ 接近随机，不收敛
v2LLM Judge~75%✅ 稳定学习，追平 GT
发现 4：反馈机制是瓶颈，不是 DGD

Embedding 相似度（50% 准确率）无法驱动学习。LLM Judge（75% 准确率）就足够了。DGD 本身在任何准确率 >60% 的反馈下都能学习。
