α<1.0 导致"死亡谷"

α=0.95 时 P@1 从 78.6% 崩到 7.1%。原因：对角线指数衰减（0.95^280≈0），基础结构崩塌。但 α=1.0 并非"不遗忘"——DGD 的 -η·k·kᵀ 项提供干扰遗忘（新学习覆盖旧学习），与"遗忘是压缩副产品"的理论一致。

α=0.95 比 α=0.9 更差（非单调）

0.95 衰减够慢让错误积累，又够快让基础崩塌——"死亡谷"最底部。0.9 衰减更快反而把错误也冲掉了一些。

累积阈值挡住了错误反馈，也挡住了正确反馈。DGD 的干扰遗忘（-η·k·kᵀ）自然处理噪声——70% 正确 vs 30% 错误，正确方向的累积力量大于错误方向。反馈准确率 >50% 时全收就行。

理论发现

DGD 的两种遗忘

M_new = M_old × (αI − η·k·kᵀ) − η·error·kᵀ

α 时间衰减（α<1.0）：所有权重每步缩小，不管有没有新学习 → 有害

干扰遗忘（-η·k·kᵀ）：只有被新学习触碰的方向被修改 → 有益

α=1.0 关掉时间衰减，保留干扰遗忘。与"遗忘是压缩副产品，不是时间函数"的认知科学理论一致。

M 矩阵是 cosine 的补丁

DGD 不替代向量库。query 先过 M 变换再和向量库比较。M 学的不是"所有关联"，只是"cosine 做错的那些纠偏"。所以 dim=256（65K 参数）就够——大部分检索 cosine 自己就对了。

反馈准确率 50% 是分水岭

BigBang/TheOffice：反馈准确率 ~70% → DGD 有效
Friends：反馈准确率 ~30% → DGD 无效
过滤噪声（累积阈值）不如全收——因为过滤也挡住了正确反馈。

最优架构

写入：对话中 → 实时写入输入层 → 立刻可检索（被动+主动） 巩固：离线定期 → 压缩合并 → 概念层（遗忘在这里产生） 检索：query → 单个 M (α=1.0) → cosine 兜底 反馈：LLM Judge 观察 → 全收不过滤 → DGD 更新 M
否定了的设计
✗ TTL（遗忘靠压缩产生，不靠时间删除）
✗ α<1.0（死亡谷，基础结构崩塌）
✗ CMS 用 α 区分层级（应用更新频率区分）
✗ 反馈累积阈值（全收更好）
✗ η=0.05（反馈有噪声时震荡太大）
✗ dim=128 或 512（256 是当前数据量的甜区）
